# MAC Phase 1.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the verified MAC MVP with observed capability metrics, automatic risk-based test planning, and a minimal HTTP adapter.

**Architecture:** Keep SQLite as the local source of truth. Observed capability metrics are recorded as append-only outcome rows and summarized for scheduling. Test planning remains a pure function that creates `TestContract` from risk signals. HTTP is a thin FastAPI adapter over `Registry`, with no Cloud Bridge, Redis, Postgres, or gRPC.

**Tech Stack:** Python 3.13 in this workspace, Pydantic v2, sqlite3, pytest, FastAPI TestClient.

---

## Scope

Included:
- Observed agent outcomes: success, failure, duration, task type, error code.
- Capability score summary exposed through Registry and CLI.
- Test contract planner from changed files and explicit risk hints.
- FastAPI app factory for register, discover, task submit, accept, start, quality, complete, audit.

Excluded:
- Real model/terminal process adapters.
- WebSocket streaming beyond app factory foundation.
- Cloud Bridge, Redis, Postgres, gRPC, multi-tenant auth.

- [ ] Write `tests/test_test_planner.py` covering low, medium, high, and security/data-sensitive file signals.
- [ ] Run `python -m pytest tests/test_test_planner.py -q` and confirm it fails because `mac.testing.planner` does not exist.
- [ ] Implement `plan_test_contract(changed_files, risk_hint=None)` returning a `TestContract`.
- [ ] Keep policy simple:
  - explicit `risk_hint` wins;
  - files containing `auth`, `security`, `payment`, `migration`, or `schema` imply high risk;
  - test-only or docs-only changes imply low risk;
  - all other code changes imply medium risk.
