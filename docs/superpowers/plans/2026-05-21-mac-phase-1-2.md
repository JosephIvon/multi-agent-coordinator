# MAC Phase 1.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add capability-based task claiming so real agent adapters can pull work from MAC without human hand-routing every task.

**Architecture:** Keep task claiming local and ledger-backed. `Registry.claim_next_task()` scans proposed tasks, matches capability and optional project context, respects explicit `to_agent`, then uses existing CAS status transition to move `proposed -> accepted` and append an audit entry. CLI and HTTP expose this as thin adapters.

**Tech Stack:** Python 3.13, Pydantic v2, sqlite3, argparse, FastAPI TestClient, pytest.

---

## Scope

Included:
- SQLite task listing by status/project context.
- Registry capability-based `claim_next_task`.
- CLI `claim` command.
- HTTP `POST /agents/{agent_id}/claim` route.
- Tests for capability matching, target-agent respect, empty queue, CLI, and HTTP.

Excluded:
- Long polling / WebSocket task streaming.
- Distributed locks beyond SQLite CAS.
- Real terminal agent process execution.
- Cloud Bridge, Redis, Postgres, gRPC.
