# MAC Phase 1.3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a local Agent Adapter Loop that can claim one task, run a controlled handler, submit quality evidence, complete or fail the task, and record observed capability metrics.

**Architecture:** Keep execution local and explicit. `LocalAgentRunner` owns the lifecycle orchestration, but business work is delegated to a configured handler. A command handler is allowed only when the caller configures the command; MAC never executes commands from task payload content.

**Tech Stack:** Python 3.13, sqlite3, Pydantic v2, argparse, subprocess, pytest.

---

## Scope

Included:
- `LocalAgentRunner.run_once()` for one claim/start/execute/finish cycle.
- Handler result model with command, status, evidence, output, error code.
- Controlled subprocess handler with timeout and caller-supplied command.
- CLI `run-once` for simple local command adapters.
- Example runner script.

Excluded:
- Long-running daemon supervisor.
- Shelling out to commands embedded inside task payloads.
- Streaming logs.
- Sandboxed process isolation beyond timeout/cwd/env.
- Cloud Bridge, Redis, Postgres, gRPC.

## Task 1: Local Runner Core

- [ ] Write `tests/test_local_runner.py` for success and handler exception failure.
- [ ] Run `python -m pytest tests/test_local_runner.py -q` and confirm failure because `mac.runner.local` does not exist.

- [ ] On success: claim task, start task, submit quality result, complete task, record succeeded observed outcome.
- [ ] On handler failure: fail task with `HANDLER_ERROR`, record failed observed outcome.

## Task 2: Controlled Command Handler

- [ ] Extend `tests/test_local_runner.py` to cover `command_task_handler` success and nonzero exit.

- [ ] Accept explicit `evidence_on_success`; default to `["test_output"]`.
