# Phase 1.4 Adapter Templates Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add reusable local adapter templates on top of `LocalAgentRunner` without expanding Phase 1 into daemon, plugin, or remote worker behavior.

**Architecture:** `LocalAgentTemplate` packages one agent identity, one capability, optional project context, metadata, and a configured handler. Template factories create command and pytest adapters, then produce ordinary `LocalAgentRunner` instances for one-shot execution.

**Tech Stack:** Python dataclasses, existing Pydantic protocol models, existing SQLite registry/ledger, pytest.

---

## File Structure

- Create: `src/mac/runner/templates.py`
  - Defines `LocalAgentTemplate`, `TaskHandler`, `command_agent_template()`, `pytest_agent_template()`, and `runner_from_template()`.
- Modify: `src/mac/runner/__init__.py`
  - Exports the template API.
- Modify: `src/mac/cli.py`
  - Wires `run-once` through `command_agent_template()` while preserving existing CLI behavior.
- Modify: `examples/local_runner.py`
  - Demonstrates template-driven runner creation.
- Create: `tests/test_runner_templates.py`
  - Covers template API and safety boundaries.
- Modify: `tests/test_task_claiming.py`, `tests/test_cli_claim.py`, `tests/test_http_claim.py`
  - Locks claim ownership semantics: successful claim persists `to_agent` and `target_agent_id`.
- Modify: `README.md`, `docs/SPEC.md`
  - Documents Phase 1.4 scope and boundaries.
