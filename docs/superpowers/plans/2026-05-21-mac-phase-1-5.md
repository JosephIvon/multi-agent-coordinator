# Phase 1.5 Task Visibility Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add read-only task visibility so agents and humans can inspect candidate work before claim without introducing queue, polling, lease, daemon, or scheduler semantics.

**Architecture:** Reuse the SQLite ledger's existing `list_task_transfers()` and `get_task_transfer()` operations behind Registry service methods. CLI and HTTP call Registry, preserving a single domain implementation.

**Tech Stack:** Python, Pydantic protocol models, existing SQLite task ledger, FastAPI adapter, argparse CLI, pytest.

- [x] **Step 3: Implement**

Add:

```python
def get_task(self, task_id: str) -> TaskTransfer | None: ...
def list_tasks(
    self,
    *,
    status: str | None = None,
    capability: str | None = None,
    agent_id: str | None = None,
    project_context: str | None = None,
) -> list[TaskTransfer]: ...
```

`agent_id` includes unassigned tasks and tasks assigned to that agent; it excludes tasks assigned to other agents.
