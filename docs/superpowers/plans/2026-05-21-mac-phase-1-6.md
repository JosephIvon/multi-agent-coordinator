# Phase 1.6 Task Evidence Bundle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a read-only task evidence bundle so agents and humans can review a task handoff from one response instead of manually joining task state, quality results, audit events, and observed metrics.

**Architecture:** Add a protocol model `TaskEvidenceBundle`. Registry builds the bundle from existing SQLite ledger reads; CLI and HTTP expose the same Registry method. No new storage tables, no state mutation, no quality gate re-evaluation.

**Tech Stack:** Python, Pydantic protocol models, existing SQLite ledger, Registry service, argparse CLI, FastAPI adapter, pytest.
