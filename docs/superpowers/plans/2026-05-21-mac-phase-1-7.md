# Phase 1.7 Quality Gate Preview Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a read-only quality gate preview so agents and humans can see whether current quality evidence satisfies a task's TestContract before calling `complete_task()`.

**Architecture:** Add `QualityGatePreview` as a protocol model. Registry computes the preview from the existing task contract and quality results, reusing `evaluate_quality_gate()` for the allow/block decision. CLI and HTTP expose the same Registry method. No task mutation, no audit append, no alternate completion path.

**Tech Stack:** Python, Pydantic protocol models, existing quality gate evaluator, Registry service, argparse CLI, FastAPI adapter, pytest.
