# Phase 1.8 Task Readiness / Next Action Preview Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a read-only task readiness report so agents and humans can inspect the current recommended next action and blocking reason without mutating task state.

**Architecture:** Add `TaskReadinessReport` as a protocol model. Registry computes the report from the current task snapshot, audit trail, required capability helper, execution-agent aliases, and `QualityGatePreview` for running tasks. CLI and HTTP expose the same Registry method. No claim, accept, start, complete, fail, audit append, or status mutation.

**Tech Stack:** Python, Pydantic protocol models, Registry service, existing quality preview helper, argparse CLI, FastAPI adapter, pytest.
